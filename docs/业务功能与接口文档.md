4. 订单管理

## 4. 接口规范

### 4.1 基础说明
- Base 前缀: `/api`
- 鉴权: `Authorization: Bearer <token>`
- 请求体: `application/json`（文件上传除外）
- 前端开发代理: Vite 将 `/api` 代理到 `http://localhost:8000`

### 4.2 统一错误风格
- 典型格式: `{"error": "错误说明"}`
- 常见状态码: `400/401/403/404/500`

## 6. 关键业务流程

### 6.1 下单流程
1. 用户在购物车页选择地址
2. 调用 `create_from_cart`
3. 后端事务化校验库存并扣减库存
4. 创建 `Order + OrderItem`
5. 清空当前用户购物车

### 6.2 支付流程
1. 前端调用 `create_payment`
2. 后端锁定订单并创建/复用待支付交易
3. 支付宝下单成功后返回二维码 URL 与二维码图片
4. 前端轮询 `status` 或回调通知触发状态更新
5. 成功后订单状态置为 `paid`

### 6.3 取消订单流程
1. 仅允许取消待支付订单
2. 后端事务内回滚库存与销量
3. 订单状态更新为 `cancelled`
