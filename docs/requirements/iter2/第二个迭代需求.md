# 第二个迭代需求

> 版本：2026-04-18（统一整合版）  
> 集成分支：`iter2/main`

---

## 1. 总体规则
- 固定合并顺序：`BE-Seckill -> BE-Payment -> FE-Biz -> QA+Doc -> 主代理联调`
- 合并门禁：
  - 契约路径与字段不漂移
  - 迭代1核心链路回归通过
  - 本轮文档与测试资产同步更新

## 2. 当前已落地基线（截至 2026-04-17）

### 2.1 用户评价与互动
- 仅已购可评价、同订单项一评
- 评价内容敏感词处理（掩码过滤）
- 评价发布后直接展示（审核接口保留但不拦截）
- 评价回复能力（用户可对评价进行回复）

### 2.2 商品流量入口
- `GET /api/products/products/top-sales/` 销量排行
- `GET /api/products/products/hot-feed/` 热榜轮播
- 首页接入热榜轮播 + 销量排行展示

### 2.3 秒杀业务（已可下单）
- 秒杀活动列表、商品维度活动查询
- 秒杀预占（幂等键 + 限流 + 预占有效期）
- 基于预占创建订单（进入正常待支付流）
- 预占取消与过期清理，库存回补
- `GET /api/seckill/my-reservations/` 我的预占记录

### 2.4 秒杀后台管理
- 秒杀活动增删改查、状态切换、库存调整、价格调整
- 同一活动组支持多个秒杀商品（`group_id`）
- 预占记录管理与手工释放
- 后台操作日志与统计看板

### 2.5 性能与缓存基础
- 榜单/热榜缓存读取
- 商品变更触发 feed 版本失效（版本号缓存键方案）

## 3. 补充清单（Backlog）

### 3.1 P0（已落地，持续强化）
1. 秒杀订单超时关闭并回补库存
2. 秒杀活动状态机扩展（前后台一致）
3. 支付回调补偿机制
4. 评价敏感词命中反馈优化

### 3.2 P1（增强）
1. 我的秒杀记录页增强（失败原因+时间线）
2. 秒杀后台批量操作增强（批量调库存/价格）
3. 排行榜策略参数化（权重配置）

### 3.3 P2（规划）
1. 评价带图/商家回复增强
2. 运营位可配化与实时热榜增强

## 4. 子代理拆分与职责

### 4.1 子代理分工
#### 子代理A（BE-Seckill）
- 职责：秒杀状态机扩展、超时回收与库存回补
- 改动域：`backend/apps/seckill/*`、`backend/apps/orders/*`（必要联动）
- 交付：状态机说明、回收契约、自测边界清单

#### 子代理B（BE-Payment）
- 职责：支付回调补偿机制与可观测日志
- 改动域：`backend/apps/payment/*`
- 交付：补偿策略、重试/终止条件、一致性自测清单

#### 子代理C（FE-Biz）
- 职责：秒杀记录页、失败原因提示、状态展示联动
- 改动域：`frontend/src/views/*`、`frontend/src/api/*`
- 交付：页面流程、状态映射表、异常提示规范

#### 子代理D（QA+Doc）
- 职责：回归用例、性能基线、契约核对与面试草稿
- 改动域：`docs/*`、`docs/testing/*`
- 交付：回归清单、性能模板、契约核对记录

### 4.2 主代理职责（不可外包）
- 预占 -> 下单 -> 支付 -> 订单最终一致性联调
- 迁移顺序与跨模块回滚策略
- 冲突仲裁（字段漂移、状态语义冲突）
- 最终验收与发布口径统一

### 4.3 冲突处理规范
- 同一接口只允许单一子代理主改
- 子代理不得跨域重写他组接口语义
- 发现冲突先回主代理裁决，不直接互改

## 5. 接口契约与状态冻结

### 5.1 已落地接口
- `POST /api/seckill/reservations/{id}/expire-or-release/`
- `GET /api/seckill/my-reservations/`（含失败原因、状态时间戳）
- `POST /api/pay/reconcile/`（管理员触发补偿）
- 评价提交增强返回：`hit_sensitive_words`, `sanitized_content`, `suggestion`

### 5.2 状态字段
- 秒杀活动：`draft/preheating/online/ended/offline`
- 预占状态：沿用现有状态并按迁移规则执行

## 6. 自测与联调门禁
1. 功能检查：补充需求 P0 通过
2. 回归检查：认证、商品、购物车、订单、支付全绿
3. 一致性检查：库存、预占、订单、支付状态一致
4. 文档检查：需求文档、测试文档、面试沉淀同步

### 6.1 失败兜底
- 任一链路出现状态漂移，回退到 `iter2-baseline-20260416` 后按模块修复

## 7. 面试沉淀映射
- 秒杀状态机与超时回收
- 支付回调补偿与最终一致性
- 子代理解耦策略与冲突治理
- 评价敏感词策略（过滤、提示、误伤边界）

对应文档：`docs/interview-skill/项目面试问答-迭代沉淀.md`
