# 第一次迭代需求文档（重构版）

## 1. 迭代目标
第一迭代目标是完成“用户可下单并支付、管理员可维护商品与订单、支持本地/外网联调”的最小可用闭环。

## 2. 范围边界
- 前端：`frontend`（Vue3 + Element Plus）
- 后端：`backend`（Django + DRF）
- 运行模式：本地直连 + Nginx 反向代理 + ngrok 外网回调
- 不在本迭代范围：推荐算法、优惠券、发票、退款自动化、复杂风控

## 3. 模块划分总览

## 5. 第一迭代完成判定（DoD）
满足以下条件可判定第一迭代完成：
1. 用户端“浏览-加购-下单-支付-看订单”闭环全通。
2. 管理端可完成商品与订单履约管理。
3. 验证码链路稳定（图片验证码可见、邮箱验证码可送达）。
4. 支付回调可在外网联调成功。
5. 测试文档与发布文档齐备，并完成首版发版。
