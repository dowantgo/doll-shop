# 第一次迭代需求

## 1. 迭代目标

第一次迭代的目标是完成玩偶商城最小可用闭环，让用户可以完成：

- 注册 / 登录
- 浏览商品
- 加入购物车
- 提交订单
- 发起支付
- 查看订单

同时管理员需要具备：

- 商品管理
- 商品上下架
- 库存维护
- 订单履约维护

## 2. 需求范围

### 2.1 用户端

- 用户注册、登录、找回密码
- 图片验证码、邮箱验证码
- 商品列表、分类筛选、关键词搜索
- 商品详情
- 购物车增删改查
- 地址管理
- 创建订单、取消待支付订单
- 支付创建、支付状态更新
- 订单列表、订单详情

### 2.2 管理端

- 商品分类管理
- 商品信息管理
- 商品图片管理
- 商品上下架和库存维护
- 订单发货与物流字段维护

### 2.3 联调与发布准备

- 本地启动链路
- Nginx 反向代理
- 外网支付回调联调
- 基础测试资料与发布资料准备

## 3. 核心业务规则

### 3.1 用户与权限

- 公开接口允许未登录访问，业务接口要求 JWT 登录态
- 普通用户只能访问自己的购物车、地址、订单等数据
- 管理端接口必须由管理员访问

### 3.2 商品与购物车

- 用户端只展示上架商品
- 库存为 0 的商品允许展示，但不可加购或下单成功
- 同一用户同一商品在购物车中只保留一条记录

### 3.3 订单与库存

- 订单从购物车创建，生成订单主表与订单项快照
- 下单时需要校验地址、商品上架状态和库存
- 成功下单后扣减库存、增加销量、清理购物车
- 失败时不允许出现半成品订单

### 3.4 支付

- 仅允许对待支付订单创建支付交易
- 支付回调需要验签，并保证幂等
- 支付成功后更新订单状态和支付状态
- 超时未支付订单需要收口处理并回补库存

## 4. 非目标

第一次迭代不包含以下内容：

- 优惠券
- 秒杀
- 自动化退款
- 推荐系统
- 复杂风控
- 大规模性能压测

## 5. 验收标准

- 用户端“浏览商品 -> 购物车 -> 下单 -> 支付 -> 查看订单”闭环可用
- 管理员可完成商品维护与订单履约维护
- 支付联调链路可验证
- 测试用例、测试报告、缺陷清单可追溯
