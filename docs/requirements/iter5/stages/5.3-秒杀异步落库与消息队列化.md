# 5.3-秒杀异步落库与消息队列化

## 阶段定位

5.3 作为第五个迭代的第三阶段规划，目标是在 5.2 的 Redis 化预占基础上，继续把秒杀建单主路径从同步落库推进到异步化处理。

## 规划方向

- 秒杀预占成功后不立即同步落订单
- 通过任务队列异步物化 `SeckillReservation / Order / OrderItem`
- 为异步消费增加幂等控制和失败补偿
- 让更多热点请求停留在 Redis + 队列层，而不是直接占用数据库连接

## 当前状态

- 已完成规划
- 尚未进入正式开发与测试

## 预期价值

- 进一步降低秒杀请求对 MySQL 连接的即时占用
- 为更高并发下的秒杀链路提供更强削峰能力
- 让 Iter5 从“稳定性增强”继续演进到“更完整的高并发秒杀主链路”
