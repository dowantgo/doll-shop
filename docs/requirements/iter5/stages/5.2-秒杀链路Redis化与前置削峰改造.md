# 5.2-秒杀链路Redis化与前置削峰改造

## 阶段目标

5.2 是第五个迭代的第二阶段，重点围绕秒杀链路做 Redis 化与前置削峰改造，把热点流量尽量挡在 MySQL 之前。

## 已落地范围

- 秒杀入口限流
- submit token 一次性消费
- Redis 库存预扣与票据化预留
- Redis 幂等回放
- `reservation_token` 驱动建单
- 过期票据异步释放与库存回补
- 后台库存变更同步 Redis 库存桶

## 阶段结论

5.2 已完成秒杀 Redis 化主链路验证，功能和一致性目标通过，中等压力下明显改善；但在 `300` 并发口径下，系统仍受 MySQL 连接数上限约束，高压能力尚未完全收口。
