# 5.1-并发治理与交易稳定性增强

## 阶段目标

5.1 是第五个迭代的第一阶段，重点补齐商城在稳定性和成熟度方面的明显短板：

- 用户入口防刷
- 购物车并发更新
- 普通下单请求幂等
- 订单过期异步清理
- 支付轮询减压
- Refresh Token 完整续签
- 统一指标采集基础

## 已落地范围

- 登录失败锁定
- 图片验证码限流
- 邮箱验证码冷却与频率限制
- 购物车原子加购与库存校验
- `X-Idempotency-Key` 普通下单幂等
- 订单过期清理由 Celery Beat 异步化
- 支付状态查询退避轮询与 pending 缓存
- Access / Refresh Token 双 token 链路

## 阶段结论

5.1 解决的是“普通交易链路更稳、更抗重复点击、更少无效请求”的问题，它为后续 5.2 的秒杀 Redis 化改造提供了前置能力基础。
