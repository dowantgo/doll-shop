# 数据库设计文档

- 文档版本：v4.0
- 更新时间：2026-04-25
- 适用分支：`iter4-main`
- 数据库：MySQL 8+

## 1. 设计原则

1. 交易主账数据落 MySQL，缓存与限流落 Redis。
2. 下单、支付、退款、秒杀释放等关键写路径必须在事务内。
3. 金额字段统一 `Decimal(10,2)`。
4. 关键状态字段可回放、可审计、可补偿。
5. Iter4 新增索引必须服务于明确查询路径，避免无目的堆索引。

## 2. 核心关系（Iter3）

1. `users_user` 1:N `orders_order`
2. `orders_order` 1:N `orders_orderitem`
3. `orders_order` 1:1 `payment_paymenttransaction`
4. `orders_order` 1:1 `coupons_orderdiscountsnapshot`（可空）
5. `users_user` 1:N `coupons_usercoupon`
6. `coupons_coupontemplate` 1:N `coupons_usercoupon`
7. `orders_orderitem` 1:N `refunds_refundrequest`
8. `refunds_refundrequest` 1:N `refunds_refundauditlog`

## 3. Iter3 新增核心表

## 3.1 `coupons_coupontemplate`
用途：优惠券模板。

关键字段：
1. `name`
2. `coupon_type`（当前为 `fixed`）
3. `discount_amount`
4. `min_spend_amount`
5. `total_quota`
6. `claimed_count`
7. `per_user_limit`
8. `valid_from`、`valid_to`
9. `status`（`active/inactive`）
10. `created_by`

索引：
1. `(status, valid_from, valid_to)`

## 3.2 `coupons_usercoupon`
用途：用户持有的券实例。

关键字段：
1. `coupon_no`（唯一）
2. `user_id`
3. `template_id`
4. `status`（`unused/locked/used/expired`）
5. `claimed_at`、`locked_at`、`used_at`

索引与约束：
1. `(user, status)`
2. `(template, status)`
3. 唯一：`coupon_no`

## 3.3 `coupons_orderdiscountsnapshot`
用途：订单优惠快照（防止规则变更后金额不可追溯）。

关键字段：
1. `order_id`（一对一）
2. `user_coupon_id`（可空）
3. `subtotal_amount`
4. `full_reduction_amount`
5. `coupon_discount_amount`
6. `final_payable_amount`
7. `pricing_version`
8. `pricing_snapshot`（JSON）

## 3.4 `refunds_refundrequest`
用途：退款申请主表。

关键字段：
1. `refund_no`（唯一）
2. `user_id`
3. `order_id`
4. `order_item_id`
5. `quantity`
6. `requested_amount`
7. `approved_amount`
8. `status`（`pending/approved/rejected/refunding/success/failed`）
9. `reason`、`review_note`、`last_error`
10. `reviewed_by`、`reviewed_at`、`processed_at`
11. `idempotency_key`
12. `external_refund_no`（预留）

索引：
1. `(user, -created_at)`
2. `(order, -created_at)`
3. `(status, -created_at)`
4. `(idempotency_key)`
5. Iter4：`(order_item, status)` 用于退款可退数量/金额聚合
6. Iter4：`(user, idempotency_key)` 用于用户维度幂等查询

## 3.5 `refunds_refundauditlog`
用途：退款审核与处理日志。

关键字段：
1. `refund_id`
2. `operator_id`
3. `action`（`create/approve/reject` 等）
4. `note`
5. `before_data`（JSON）
6. `after_data`（JSON）
7. `created_at`

## 4. 现有表在 Iter3 的关键扩展

## 4.1 `payment_paymenttransaction`
新增补偿审计字段：
1. `reconcile_attempts`
2. `last_reconcile_at`
3. `last_reconcile_status`
4. `last_reconcile_error`

## 4.2 `orders_order`
关键交易/履约字段：
1. `payment_status`、`payment_method`、`paid_at`
2. `shipping_company`、`shipping_status`、`tracking_no`
3. `shipped_at`、`delivered_at`
4. `expires_at`

## 5. Redis 使用（与 Iter3 相关）

1. `products:feed:version`：商品 feed 缓存版本。
2. `ratelimit:seckill_pre_reserve:*`：秒杀预占限流。
3. 会话、验证码、排行榜等短期缓存。

说明：优惠、退款、支付补偿结果均以 MySQL 为准，Redis 不作交易主账。

## 6. 数据一致性策略

1. 订单锁券与解券在事务中处理。
2. 支付成功后同步更新订单与券状态。
3. 退款申请/审核使用幂等键和状态机防重入。
4. 补偿任务以“幂等收敛”为目标，可重复执行。
