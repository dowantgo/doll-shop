# 数据库设计文档

- 文档版本: v2.0
- 更新时间: 2026-04-18
- 数据库: MySQL 8+
- ORM: Django ORM

## 1. 设计原则
1. 业务主数据落 MySQL，缓存和验证码落 Redis。
2. 交易链路（下单/支付/秒杀释放）使用事务保障一致性。
3. 高频查询字段建立索引，优先覆盖“订单、商品、秒杀、评价”路径。
4. 状态机字段可枚举、可追踪、可补偿。

## 3. 主要数据表说明

## 3.1 用户与地址

### users_user
- 用途：账号、角色、认证主体。
- 关键字段：`username`(唯一)、`email`、`phone`、`role(user/admin)`、`created_at`。

### users_address
- 用途：收货地址。
- 关键字段：`user_id`、`province/city/district/address`、`is_default`。

## 3.2 商品域

### products_category
- 用途：商品分类。
- 关键字段：`name`(唯一)、`sort_order`。

### products_product
- 用途：商品主表。
- 关键字段：`name`、`category_id`、`price`、`stock`、`sales`、`is_hot`、`status`。
- 关键索引：`status`、`(is_hot, hot_sort_order)`。

### products_productimage
- 用途：商品图片（支持主图）。
- 关键字段：`product_id`、`image`、`is_main`、`sort_order`。

### products_inventorylog
- 用途：库存变更审计。
- 关键字段：`product_id`、`log_type(in/out/adjust/return)`、`before_stock`、`after_stock`。

## 3.3 交易域

### cart_cartitem
- 用途：购物车项。
- 关键字段：`user_id`、`product_id`、`quantity`。
- 约束：联合唯一 `(user_id, product_id)`。

### orders_order
- 用途：订单主表。
- 关键字段：
  - 业务号：`order_id`(唯一)
  - 状态：`status(pending/paid/shipped/delivered/cancelled)`
  - 支付状态：`payment_status(pending/paid)`
  - 支付方式：`payment_method(alipay/wechat/mock)`
  - 物流：`shipping_company`、`shipping_status`、`tracking_no`
  - 超时：`expires_at`
- 索引：`(user, -created_at)`、`status`、`payment_status`。

### orders_orderitem
- 用途：订单商品快照。
- 关键字段：`order_id`、`product_id`、`quantity`、`price`。

### payment_transaction
- 用途：支付交易单（当前唯一主支付账务模型）。
- 关键字段：
  - `order_id`
  - `out_trade_no`(唯一)
  - `trade_no`(唯一，可空)
  - `amount`
  - `status(pending/paid/expired/closed/failed/refunded)`
  - `qr_code`、`expire_time`、`paid_time`
- 第二迭代补偿字段：
  - `reconcile_attempts`
  - `last_reconcile_at`
  - `last_reconcile_status`
  - `last_reconcile_error`

## 3.4 评价域（第二迭代新增）

### reviews_review
- 用途：用户评价。
- 关键字段：
  - `order_item_id`（OneToOne，确保同订单项一评）
  - `product_id`、`user_id`
  - `rating`、`content`
  - `status(pending/approved/rejected)`（当前默认 approved）
  - 审核扩展：`audit_by`、`audit_at`、`audit_remark`
- 索引：
  - `(product, status, -created_at)`
  - `(user, -created_at)`
  - `(status, -created_at)`

### reviews_reviewreply
- 用途：评价回复（用户对评价进行互动回复）。
- 关键字段：`review_id`、`user_id`、`content`、`created_at`。
- 索引：`(review, created_at)`、`(user, -created_at)`。

## 3.5 秒杀域（第二迭代新增）

### seckill_activity
- 用途：秒杀活动定义（支持同组多商品）。
- 关键字段：
  - `group_id`（活动分组 ID）
  - `product_id`
  - `seckill_price`
  - `total_stock`、`reserved_stock`
  - `per_user_limit`
  - `status(draft/preheating/online/ended/offline)`
  - `is_enabled`、`start_at`、`end_at`
- 索引：`(status, is_enabled, start_at)`、`(product, -created_at)`。

### seckill_reservation
- 用途：秒杀预占记录（库存锁定中间态）。
- 关键字段：
  - `activity_id`、`product_id`、`user_id`
  - `quantity`
  - `status(reserved/ordered/paid/cancelled/expired)`
  - `idempotency_key`
  - `order_id`（可空，OneToOne）
  - `reserved_expires_at`
- 索引：`(activity, status, -created_at)`、`(user, -created_at)`。

### seckill_admin_action_log
- 用途：秒杀后台操作审计日志。
- 关键字段：`operator_id`、`activity_id`、`reservation_id`、`action_type`、`before_data`、`after_data`、`remark`。
- 索引：`(action_type, -created_at)`、`(operator, -created_at)`、`(activity, -created_at)`。

## 4. 事务与一致性

### 4.1 下单事务
1. 校验地址与库存
2. 锁定商品行（`select_for_update`）
3. 写入 `orders_order`、`orders_orderitem`
4. 扣减库存与更新销量

### 4.2 支付事务
1. 锁定订单/支付单
2. 更新 `payment_transaction`
3. 同步更新 `orders_order.payment_status/status`
4. 秒杀订单时同步 `seckill_reservation.status`

### 4.3 秒杀释放事务
1. 锁定预占记录与活动
2. 回收 `reserved_stock`
3. 必要时回补商品库存
4. 更新预占状态为 `expired/cancelled`

## 5. 缓存数据（Redis）

### 5.1 业务缓存
- 产品 feed 版本键：`products:feed:version`
- 榜单缓存键：`products:feed:{feed}:v{version}:limit:{n}`

### 5.2 会话与验证码
- Session：Django cache session backend
- 图片验证码：`captcha:*`
- 邮箱验证码：`email_code:*`

## 6. 迁移与维护建议
1. 所有结构变更通过 Django migration 管理。
2. 生产执行迁移前先备份数据库。
3. 对 `payment_transaction` 与 `seckill_reservation` 建立定期数据归档策略。
4. 补充数据库级巡检：
   - 超时未关闭订单
   - 秒杀保留库存与实际预占不一致
   - 支付单状态与订单状态不一致
