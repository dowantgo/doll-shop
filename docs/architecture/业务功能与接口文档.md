## 1. 功能总览

## 1.1 用户端功能

1. 图片验证码登录
2. 邮箱验证码注册

4. JWT 鉴权

### B. 商品浏览（已实现）

2. 关键词搜索
3. 商品详情
4. 热榜轮播与销量排行（第二迭代新增）

### C. 购物车与下单（已实现）

### F. 秒杀专区（第二迭代已实现）
1. 秒杀活动列表
2. 秒杀预占
3. 基于预占创建订单
4. 我的预占记录
5. 预占取消与释放

## 2. 角色与权限

### 2.1 角色
- `user`: 普通用户
- `admin`: 管理员

### 2.2 权限规则
1. 公共接口：验证码、注册、登录、商品浏览相关

## 4. 接口规范

- Base URL 前缀：`/api`
- 鉴权：`Authorization: Bearer <token>`
- 默认请求体：`application/json`（文件上传除外）
- 常见错误格式：`{"error": "错误描述"}`

## 6. 关键业务流程

### 6.1 常规下单流程
1. 用户提交购物车创建订单
2. 后端事务校验库存并扣减
3. 生成订单和订单项
4. 返回待支付订单

### 6.2 支付流程
1. 创建支付单，返回二维码 URL 与二维码图片
2. 用户扫码支付
3. 回调或轮询更新支付状态
4. 异常场景可触发 `reconcile` 补偿

### 6.3 秒杀流程
1. 查询活动并发起预占
2. 预占成功后创建秒杀订单
3. 秒杀订单进入支付链路
4. 成功后更新预占为 paid，失败/超时释放库存

### 6.4 评价流程
1. 已购用户提交评价
2. 后端敏感词处理并返回反馈字段
3. 评价直接展示
4. 其他用户可回复评价
