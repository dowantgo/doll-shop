# 业务功能与接口文档

- 文档版本：v4.0
- 更新时间：2026-04-25
- 适用分支：`iter4-main`

## 1. 功能总览

### 1.1 用户端
1. 账号体系：注册、登录、找回密码、图形验证码、邮箱验证码。
2. 商品浏览：分类筛选、关键词搜索、热榜轮播、销量排行。
3. 购物车：加入、改数量、移除、清空。
4. 下单与支付：购物车下单、订单详情、支付创建、支付状态查询、取消订单。
5. 秒杀：秒杀专区、预占、秒杀下单、预占记录。
6. 评价：已购可评、敏感词过滤、评价回复。
7. Iter3 新增：
   - 优惠券领取与试算；
   - 待支付订单应用优惠券；
   - 部分退款申请与进度查看；
   - 订单物流轨迹查看。
8. Iter4 性能治理：
   - 商品热榜/销量榜缓存命中统计；
   - 交易核心接口查询优化；
   - Service 分层，不改变用户接口。

### 1.2 管理端
1. 商品、分类、用户、订单管理。
2. 秒杀活动管理（含多商品同组活动）。
3. Iter3 新增：
   - 优惠券模板管理与发券；
   - 退款审核台（同意/拒绝）；
   - 手工支付补偿入口。

## 2. 统一约定

1. API 前缀：`/api`
2. 鉴权：`Authorization: Bearer <token>`
3. 时间：`Asia/Shanghai`
4. 金额精度：统一保留 2 位小数
5. 新接口优先走 Iter3 标准路径，同时保留部分旧路径别名用于兼容。
6. Iter4 不新增用户侧业务 API，性能治理不改变请求参数和响应结构。

## 4. Iter3 关键业务规则

### 4.1 优惠规则
1. 单订单最多使用 1 张用户券。
2. 计算顺序：先满减，再券抵扣。
3. 订单优惠快照落库，支持复算与审计。

### 4.2 退款规则
1. 仅已支付订单可申请退款。
2. 按订单项与数量退款。
3. 退款金额按订单项实付分摊。
4. 退款状态机：`pending -> approved/rejected -> refunding -> success/failed`。

### 4.3 支付补偿规则
1. 支持管理员手工补偿。
2. 支持每 15 分钟自动补偿任务。
3. 补偿记录包括次数、最后状态、最后错误、最后补偿时间。

### 4.4 物流规则
1. 未发货返回空轨迹与提示。
2. 通过环境变量切换 `mock` 与 `kuaidi100` provider。
3. 第三方失败降级为“暂无法获取轨迹”，不阻断订单主流程。
