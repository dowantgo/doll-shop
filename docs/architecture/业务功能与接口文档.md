# 业务功能与接口文档

- 文档版本：v4.1
- 更新时间：2026-04-26
- 适用分支：`iter4-main`

## 1. 功能总览

### 1.1 用户端
1. 账号体系：注册、登录、找回密码、图片验证码、邮箱验证码
2. 商品浏览：分类筛选、搜索、热榜轮播、销量排行
3. 购物车：加入、改数量、移除、清空
4. 订单与支付：下单、订单详情、支付创建、支付状态查询、取消订单
5. 秒杀：秒杀专区、预占、秒杀下单、预占记录
6. 评价：已购可评、敏感词过滤、评价回复
7. 优惠券：领取、价格试算、应用到待支付订单
8. 退款：按订单项和数量发起部分退款、查看进度
9. 物流：查询发货轨迹与时间线

### 1.2 管理端
1. 商品、分类、用户、订单管理
2. 秒杀活动管理（支持同组活动与多商品）
3. 优惠券模板管理与发放
4. 退款审核台
5. 支付补偿入口
6. 后台发货与物流状态维护

## 2. 统一约定

1. API 前缀：`/api`
2. 鉴权：`Authorization: Bearer <token>`
3. 金额精度：统一保留 2 位小数
4. 时区：`Asia/Shanghai`
5. Iter4 不新增普通用户业务 API，性能治理不改变接口路径和响应结构
6. 保留部分兼容别名路径，确保旧前端和历史测试脚本可继续运行

## 4. 关键业务规则

### 4.1 优惠规则
1. 单订单最多使用 1 张用户券
2. 计算顺序：先满减，再券抵扣
3. 订单优惠快照落库，支持复算与审计

### 4.2 退款规则
1. 仅已支付订单可申请退款
2. 按订单项与数量退款
3. 退款金额按订单项实付比例分摊
4. 退款状态机：`pending -> approved/rejected -> refunding -> success/failed`

### 4.3 支付补偿规则
1. 支持管理员手工补偿
2. 支持每 15 分钟自动补偿任务
3. 支付单记录补偿次数、最后状态、最后错误、最后补偿时间

### 4.4 物流规则
1. 未发货返回空轨迹与提示
2. `LOGISTICS_PROVIDER` 可切换 `mock` / `kuaidi100`
3. 第三方失败时降级提示，不阻塞订单主流程
