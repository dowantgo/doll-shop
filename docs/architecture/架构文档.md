# 架构文档

- 文档版本: v2.0
- 更新时间: 2026-04-18
- 适用分支: `iter2/main`

## 1. 架构概览

项目采用前后端分离架构：
- 前端：Vue 3 + Vite + Element Plus
- 后端：Django + Django REST Framework
- 数据层：MySQL（业务主存储）+ Redis（缓存/会话/验证码）
- 支付：支付宝（主流程）+ mock（本地联调）

```text
Browser
  -> Frontend (Vue3)
  -> /api/* (Axios)
  -> Django REST API
  -> MySQL (orders/products/seckill/reviews/payment)
  -> Redis (feed cache/version, captcha/email code, session)
  -> Alipay Gateway (create/query/notify)
```

## 2. 后端模块划分

### 2.1 核心业务应用
- `apps/users`：认证、图片验证码、邮箱验证码、地址管理
- `apps/products`：分类、商品、商品图片、库存日志、榜单/热榜缓存
- `apps/cart`：购物车读写
- `apps/orders`：下单、取消、发货、收货、超时关单清理
- `apps/payment`：支付单创建、状态查询、关闭、异步回调、对账补偿
- `apps/reviews`：已购评价、敏感词过滤、评价回复
- `apps/seckill`：秒杀活动、预占、秒杀下单、预占释放、后台运营管理

### 2.2 路由入口
- 总入口：`backend/dollshop/urls.py`
- 主要前缀：
  - `/api/users/`
  - `/api/products/`
  - `/api/reviews/`
  - `/api/cart/`
  - `/api/orders/`
  - `/api/seckill/`
  - `/api/admin/`
  - `/api/pay/`

## 3. 第二迭代新增能力（架构层）

### 3.1 秒杀链路（预占 + 下单 + 支付）
新增模块：`apps/seckill`，核心思路是“先预占再下单”：
1. 用户请求秒杀预占（幂等键、限购、活动状态校验）
2. 预占成功后生成秒杀订单
3. 订单进入统一支付链路
4. 支付成功后同步秒杀预占状态
5. 超时/取消触发释放与库存回补

### 3.2 评价互动链路
新增模块：`apps/reviews`：
1. 基于订单项判断是否“已购可评”
2. 提交评价时执行敏感词处理
3. 返回命中反馈字段（便于前端提示）
4. 支持对评价进行回复

### 3.3 支付补偿链路
在支付模块新增补偿接口：
- `POST /api/pay/reconcile/`
- 用于处理“回调异常、状态未落库”的支付单补偿
- 通过补偿尝试更新 `PaymentTransaction` 与 `Order` 的最终一致性

### 3.4 首页榜单缓存
在商品模块新增 feed 版本缓存策略：
- `products:feed:version` 作为版本号键
- 热榜/销量榜缓存键挂版本号
- 商品变更通过 `on_commit` 递增版本号触发失效

## 4. 关键调用链

### 4.1 常规下单链路
1. 购物车提交地址创建订单
2. 后端事务扣减库存并写入订单项
3. 清理购物车
4. 返回订单详情

### 4.2 支付链路
1. 创建支付单（锁定订单）
2. 返回 `qr_code` + `qr_code_image(base64)`
3. 前端轮询支付状态 / 接收异步回调
4. 更新交易单与订单状态
5. 必要时执行 `reconcile` 补偿

### 4.3 秒杀链路
1. 活动查询（按商品或活动列表）
2. 预占（控制库存与限购）
3. 秒杀下单（绑定预占记录）
4. 支付成功后预占状态转 paid
5. 失败/超时触发释放与回补

## 5. 数据一致性设计

### 5.1 事务边界
- 下单、取消、支付落库、秒杀释放均在事务中执行
- 高并发写路径使用 `select_for_update`

### 5.2 状态一致性
- 订单：`pending -> paid -> shipped -> delivered`，异常可 `cancelled`
- 支付单：`pending -> paid/closed/expired/failed`
- 秒杀预占：`reserved -> ordered -> paid`，异常到 `cancelled/expired`

### 5.3 超时与清理
- 订单超时依据 `expires_at` 清理并关闭
- 秒杀预占依据 `reserved_expires_at` 清理并释放配额

## 6. 缓存与会话

- Redis `default` cache backend（`django-redis`）
- 会话引擎：`django.contrib.sessions.backends.cache`
- 验证码缓存：
  - 图片验证码：`captcha:*`
  - 邮箱验证码：`email_code:*`
- 榜单缓存：
  - 版本键：`products:feed:version`
  - 业务键：`products:feed:{feed_name}:v{version}:limit:{n}`

## 7. 部署与联调模式

### 7.1 本地开发
- 前端：`localhost:5173`
- 后端：`localhost:8000`
- Vite 代理 `/api` 到后端

### 7.2 外网回调联调
- 使用 ngrok/Nginx 暴露后端回调地址
- 将 `ALIPAY_NOTIFY_URL` 配置为公网可访问地址
