# 第四个迭代测试报告

- 版本：Iter4
- 测试对象：性能优化与架构治理专项
- 依据文档：`docs/requirements/iter4/第四个迭代性能优化与架构治理.md`

## 1. 测试范围

本轮不新增大型业务功能，主要验证：

- Redis feed 缓存治理效果
- 高热点接口性能优化效果
- Service 分层后的接口兼容性
- Iter3 主交易闭环回归

## 2. 测试执行摘要

| 指标 | 说明 |
| --- | --- |
| 自动化回归 | `16/16` 通过 |
| 核心 smoke | 登录、商品、购物车、订单、支付、秒杀、优惠券、退款、物流、后台发货全绿 |
| 性能对比 | Iter4 相比 Iter3 已无回退项 |
| 发布结论 | Iter4 可作为主干性能治理版本 |

## 3. 关键性能结果

| 接口 | Iter3 P95 | Iter4 最终 P95 | 变化 |
| --- | ---: | ---: | ---: |
| `top-sales` | 566.02ms | 328.51ms | 下降 41.96% |
| `hot-feed` | 511.33ms | 272.78ms | 下降 46.65% |
| `refund-create` | 557.44ms | 441.51ms | 下降 20.80% |
| `logistics-query` | 366.26ms | 348.76ms | 下降 4.78% |
| `price-preview` | 716.53ms | 537.94ms | 下降 24.92% |

## 4. 关键验证点

1. Redis feed 缓存策略已生效，热榜与销量排行稳定可用。
2. `price-preview` 第一轮回退问题已通过探针缓存化方案修复。
3. Service 分层后前后端接口未发生结构性漂移。
4. Iter3 的优惠券、退款、物流和支付补偿主链路在 Iter4 下仍保持稳定。

## 5. 测试结论

第四个迭代已经完成性能与架构治理目标，达到“功能回归通过、性能验收通过、可合入主干”的标准。该版本可作为后续 Iter5 稳定性增强的性能基线。
