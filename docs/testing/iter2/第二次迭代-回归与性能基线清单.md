# 第二次迭代回归与性能基线清单

## 1. 文档目标
- 给出 Iter2 的最小可执行回归清单
- 单独固化接口回归点（冻结契约）
- 单独固化性能指标口径（P95、缓存命中率、首屏耗时）
- 严格区分 `已实现` 与 `规划中`

## 4. 性能基线指标与采集口径

## 4.1 指标定义（全部已实现采集）
- P95（接口）：
  - `GET /api/products/products/top-sales/`
  - `GET /api/products/products/hot-feed/`
- 缓存命中率：
  - 榜单/热榜缓存命中率（命中次数 / 总请求次数）
- 首页首屏耗时：
  - 首屏关键内容可见耗时（建议记录 LCP 与首屏业务数据渲染完成时间）

## 4.2 采集建议
- 样本：每接口至少 50 次请求（冷启动1次 + 热请求49次）
- 环境固定：
  - 相同机器、相同网络、相同数据集
  - 测试前清理无关后台任务
- 记录维度：
  - 时间戳、分支、提交哈希、环境、样本数、P50/P95、失败率

## 4.4 秒杀链路性能观测（新增）
- 状态：`已上线（持续优化）`
- 观测目标：
  - 限流触发率
  - 幂等重复请求比率
  - 预占接口 P95
  - 预占失败原因分布（限流/库存不足/重复提交）
- 通过口径：
  - 可稳定触发限流与幂等
  - 并发下无超卖、无脏状态

## 5. 执行清单（建议顺序）

1. 执行 Iter2 增量功能回归（评价、排行、轮播）
2. 执行接口回归点 `I2-API-01~06`
3. 执行管理员评价审核专项（列表筛选、通过/驳回、权限、边界）
4. 执行秒杀预埋骨架专项（活动、预占、幂等、限流、支付边界）
5. 执行迭代1核心链路回归
6. 采集性能基线（P95、命中率、首屏耗时）
7. 输出结论：通过/阻塞/风险清单

## 6. 出口准则（发布门禁）
- 必须满足：
  - Iter2 增量功能回归通过
  - 接口回归点全通过
  - 迭代1核心链路全通过
  - 性能基线数据齐全，可对比
- 可带风险发布（需备注）：
  - `未上线` 或 `预埋` 项（管理端审核页面、秒杀正式交易链路）不影响已实现功能



## 7. 本轮 P0 补充回归项

### 7.1 秒杀状态机
- 验证活动状态是否按文档口径展示。
- 验证前后台状态是否一致。
- 验证终态是否禁止回退。

### 7.2 秒杀超时回收
- 验证预占 TTL 到期后自动释放。
- 验证库存回补后可售数量恢复。
- 验证并发释放时不会重复回补。

### 7.3 支付补偿
- 验证支付回调失败后的补偿修复。
- 验证补偿重试幂等性。
- 验证最终订单状态与支付状态一致。

### 7.4 敏感词反馈
- 验证敏感词命中提示是否返回。
- 验证清洗后的内容与前端提示是否一致。
- 验证评价回复链路不受敏感词反馈影响。
