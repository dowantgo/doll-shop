# 第二次迭代-缺陷清单

## 1. 缺陷整理口径

本清单按根因合并，用于保留 Iter2 阶段的代表性问题和发布阻塞项。

## 2. 主要缺陷

### BUG-ITER2-001 后台评价审核接口整组缺失

- 严重程度：高
- 影响：阻塞评价审核链路
- 现象：`GET /api/admin/reviews/` 返回 `404`
- 根因：后台审核视图存在，但路由未注册

### BUG-ITER2-002 已取消待支付订单仍可继续 mock_pay

- 严重程度：高
- 影响：订单状态和支付状态漂移
- 现象：订单已取消后，关联支付单仍可被 `mock_pay` 标记成功

### BUG-ITER2-003 普通用户可读取其他用户详情

- 严重程度：高
- 影响：对象级数据隔离失效
- 现象：普通用户可读取其他用户详情接口数据

### BUG-ITER2-004 后台商品更新后 feed 缓存不失效

- 严重程度：中
- 影响：热榜展示与后台商品数据不一致
- 现象：后台更新商品后，`hot-feed` 仍返回旧价格

## 3. 阶段结论

Iter2 说明项目已经进入“复杂业务 + 缓存一致性 + 权限细粒度控制”的阶段。虽然主能力基本可用，但这一轮也暴露出对象级权限、状态收口和缓存失效这类更接近真实项目的问题。
