## 1. 生成口径

- 用例字段统一为：`用例编号 / 模块名称 / 用例标题 / 优先级 / 关联需求ID / 设计方法 / 前置条件 / 测试步骤 / 预期结果 / 标签 / 回归类型 / 实现状态 / 备注`
- 优先级口径：
  - `P0`：阻塞发布的核心主链路或一致性能力
  - `P1`：核心回归必测项
  - `P2`：异常、边界、扩展和预埋专项
  - `P3`：本轮未单独展开，避免过度稀释主测试集
- 回归类型口径：
  - `冒烟`：提测即跑
  - `核心`：每日/每轮提测执行
  - `全量`：发版前或专项回归执行
