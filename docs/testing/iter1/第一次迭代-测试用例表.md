# 第一次迭代-测试用例表

## 1. 覆盖目标

第一次迭代测试主要覆盖商城最小可用闭环：

- 注册 / 登录
- 商品浏览与搜索
- 购物车
- 地址管理
- 订单创建、取消、超时清理
- 支付创建、支付成功、支付回调
- 管理端商品与物流维护

## 2. 核心测试模块

| 模块 | 重点验证内容 |
| --- | --- |
| 用户认证 | 注册、登录、验证码、鉴权、权限拦截 |
| 商品 | 上下架、分类筛选、搜索、详情、图片展示 |
| 购物车 | 加购、数量修改、删除、清空 |
| 订单 | 下单、库存扣减、价格快照、取消、超时 |
| 支付 | 创建支付、状态查询、回调幂等、mock 支付 |
| 管理端 | 商品 CRUD、图片管理、发货与物流维护 |

## 3. 用例执行要求

- P0：主链路冒烟，提测和发布前必须通过
- P1：核心回归，覆盖库存、订单、支付、权限等高风险点
- P2：联调、兼容、边界与配置校验
- P3：轻量性能基线和发布资料检查
