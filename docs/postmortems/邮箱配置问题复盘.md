# 邮箱配置问题复盘（最终版）

## 1. 结论先行
本次“验证码收不到”的主因是 **Django 进程实际加载到的邮箱配置与我们以为的不一致**，而不是业务代码发信逻辑本身。

你已经确认在修复时是“正规 IP/正常网络”，因此这份最终结论聚焦在配置加载链路：
- IDE里看到的配置 ≠ Django进程内真实配置
- .env 文件内容、load_dotenv 调用方式、进程重启时机共同决定最终值

---

## 2. 现象
- 前端提示“验证码已发送”或发验证码失败
- 手工 smtplib 测试某组账号可发
- 但 Django 发信报错或读取到另一组邮箱账号

这说明问题在“配置被谁读取、何时读取、读的是哪份”。

---

## 3. 底层原因（重点）

### 3.1 python-dotenv 的加载不是“全局唯一真相”
load_dotenv() 只是把 .env 内容写入当前 Python 进程的 os.environ。关键点：

1. 默认不覆盖已有环境变量
- load_dotenv(override=False) 是默认行为
- 如果系统环境里已经有 EMAIL_HOST_USER，.env 同名项可能不会生效

2. 读取路径可能有歧义
- 不显式传路径时，会按当前执行上下文寻找 .env
- 不同启动目录/工具链可能读到不同位置

3. 读取时机只在进程启动阶段
- .env 改完不重启 Django，旧进程仍然用旧值

所以“我改了 .env 但服务仍用旧邮箱”在机制上完全可能发生。

---

### 3.2 为什么会出现“IDE看到A，Django用B”
这是常见误区，不是 IDE 一定坏，而是“观察点不一致”：
- IDE看到的是“某个文件当前内容”（甚至可能是未保存缓冲区）
- Django用的是“进程内 os.environ 的最终值”

只看文件，不看进程内值，就会误判。

推荐以进程内值为准：
`python
from django.conf import settings
print(settings.EMAIL_HOST_USER)
`

---

### 3.3 Django unserver 的重载机制会放大误解
开发环境常见：
- 你改了 .env
- 代码可能触发/不触发重载
- 子进程是否重启、何时重启并不总是直觉一致

因此邮件配置这种“启动期读取”的参数，最稳妥做法是：
- 改 .env 后手动完整重启 Django
- 必要时用 --noreload 做一次排查，避免多进程干扰

---

## 4. 最终修复动作
1. 明确 dotenv 路径（消除路径歧义）
`python
BASE_DIR = Path(__file__).resolve().parent.parent
load_dotenv(BASE_DIR / '.env')
`

2. 核对 ackend/.env 的真实内容（不用“猜”）
`powershell
Get-Content d:\shop\backend\.env
`

3. 改完配置后完整重启 Django
`powershell
cd d:\shop\backend
.\venv\Scripts\python.exe manage.py runserver 0.0.0.0:8000 --noreload
`

4. 以 Django 进程内值验收
`python
from django.conf import settings
print(settings.EMAIL_HOST, settings.EMAIL_PORT, settings.EMAIL_HOST_USER)
`

---

## 5. 这次问题的准确定性
- 不是“邮件发送逻辑写错”
- 不是“用户邮箱不支持”
- 核心是：配置加载链路与认知不一致（文件视角 vs 进程视角）

换句话说：
> 你以为改的是“系统正在使用的配置”，
> 但服务进程实际上还在用旧环境值或另一份配置。

---

## 6. 防复发清单（可执行）
1. settings.py 统一显式 load_dotenv(BASE_DIR / '.env')
2. 统一只维护 ackend/.env，避免多份 .env 并存
3. 每次改 SMTP 配置后都重启 Django
4. 验收时先看 settings.*，再测发信，不只看 IDE 文件
5. 文档与日志中不出现明文授权码（出现过就立即重置）

## 8. 30秒定位模板（下次直接套用）

### 8.1 目标
用最短路径判断“是配置未生效、还是SMTP不可达、还是认证失败”。

### 8.2 30秒命令
1) 看后端实际读取值（不是看文件）
`powershell
cd d:\shop\backend
.\venv\Scripts\python.exe manage.py shell -c "from django.conf import settings;print(settings.EMAIL_HOST, settings.EMAIL_PORT, settings.EMAIL_USE_SSL, settings.EMAIL_USE_TLS, settings.EMAIL_HOST_USER, settings.DEFAULT_FROM_EMAIL)"
`

2) 看 .env 真实内容
`powershell
Get-Content d:\shop\backend\.env
`

3) 先测网络端口
`powershell
Test-NetConnection smtp.qq.com -Port 465
Test-NetConnection smtp.qq.com -Port 587
`

4) 最小化发信测试（绕过业务）
`powershell
cd d:\shop\backend
.\venv\Scripts\python.exe manage.py shell -c "from django.conf import settings;from django.core.mail import send_mail;print(send_mail('SMTP测试','test',settings.DEFAULT_FROM_EMAIL,[settings.EMAIL_HOST_USER],fail_silently=False))"
`

### 8.3 结果判读
- settings 值不对：配置加载链路问题（路径/重启/环境变量覆盖）
- 端口不通：网络层问题
- 端口通但 send_mail 失败：认证/TLS参数问题（授权码、SSL/TLS组合）

### 8.4 固化动作
- 改配置后固定重启：
`powershell
.\venv\Scripts\python.exe manage.py runserver 0.0.0.0:8000 --noreload
`
- 先看 settings.* 再看页面提示，不被“文件显示值”误导。
