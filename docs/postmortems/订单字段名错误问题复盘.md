# 订单字段名错误问题复盘

## 1. 结论先行

本次"数据概览"页面"加载数据失败"、所有统计数据均为 0 的根本原因是：

**Order 模型使用的字段名是 `total_price`，但 `admin_views.py` 中的 Stats 查询和前端模板中错误地使用了 `total_amount`**

---

## 2. 现象

- 管理员登录后访问 `/admin/dashboard`
- 页面弹出错误提示："加载数据失败"
- 商品总数、订单总数、用户总数、总销售额均显示为 0
- 后端报错：`FieldError: Cannot resolve keyword 'total_amount' into field`

---

## 3. 底层原因

### 3.1 模型定义与代码引用不一致

Order 模型中定义的是 `total_price` 字段：

```python
# apps/orders/models.py
class Order(models.Model):
    total_price = models.DecimalField(max_digits=10, decimal_places=2, verbose_name='总价')
```

但在 `admin_views.py` 的统计查询中错误地使用了 `total_amount`：

```python
# apps/users/admin_views.py
'total_revenue': Order.objects.filter(status__in=['paid', 'shipped', 'completed']).aggregate(
    total=Sum('total_amount')  # 错误：应该是 total_price
)['total'] or 0
```

### 3.2 Django ORM 字段敏感性

Django 的 `aggregate()` 和 `annotate()` 等查询方法严格依赖字段名：
- 字段名写错 → `FieldError` 异常
- 异常被 DRF 捕获 → 返回 500 错误
- 前端接收到 500 → 走 error 分支 → 显示"加载数据失败"

### 3.3 前端模板字段名错误

前端在两个位置使用了错误的字段名：

```vue
<!-- components/admin/OrderManageView.vue -->
<el-table-column prop="total_amount" label="金额">  <!-- 错误 -->

<!-- components/admin/DashboardView.vue -->
¥{{ formatAmount(row.total_amount) }}  <!-- 错误 -->
```

实际应使用 `total_price`。

---

## 4. 修复动作

### 4.1 后端修复

**文件**：`apps/users/admin_views.py`

```python
# 修复前
'total_revenue': Order.objects.filter(...).aggregate(
    total=Sum('total_amount')  # ❌ 错误
)['total'] or 0

# 修复后
'total_revenue': Order.objects.filter(...).aggregate(
    total=Sum('total_price')  # ✅ 正确
)['total'] or 0
```

### 4.2 前端修复

**文件 1**：`components/admin/OrderManageView.vue`

```vue
<!-- 修复前 -->
<el-table-column prop="total_amount" label="金额">
  <template #default="{ row }">¥{{ row.total_amount }}</template>

<!-- 修复后 -->
<el-table-column prop="total_price" label="金额">
  <template #default="{ row }">¥{{ row.total_price }}</template>
```

**文件 2**：`components/admin/DashboardView.vue`

```vue
<!-- 修复前 -->
<el-table-column prop="total_amount" label="金额">
  <template #default="{ row }">¥{{ formatAmount(row.total_amount) }}</template>

<!-- 修复后 -->
<el-table-column prop="total_price" label="金额">
  <template #default="{ row }">¥{{ formatAmount(row.total_price) }}</template>
```

## 5. 问题定性

| 维度 | 说明 |
|------|------|
| 问题类型 | 字段名不一致（命名不一致） |
| 影响范围 | 数据概览 API、订单列表展示、最近订单展示 |
| 严重程度 | 高（导致功能完全不可用） |
| 根因 | 多人协作或跨文件开发时，未统一确认模型字段名 |

---

## 6. 防复发清单

1. **统一字段命名规范**
   - 在项目规范中明确：前后端约定使用 `total_price` 作为订单金额字段名
   - 避免同一概念使用不同名称（如 `total_amount`、`total`、`order_amount` 等）

2. **模型字段变更时同步审查**
   - 修改模型字段后，使用 IDE 全局搜索检查所有引用位置
   - 或在 PR/MR 中要求审查者检查所有用到该模型的地方

3. **Serializer 字段审查**
   - Serializer 是前后端数据契约，应在创建时确定字段名
   - 本项目中 `OrderSerializer` 已正确使用 `total_price`，但 admin_views.py 和前端漏改

4. **后端单元测试**
   - 为 Stats API 编写单元测试，验证返回的字段名与 Serializer 一致
   - 测试应检查 `total_price` 而非 `total_amount`

---

```python

```

### 7.2 查看 Serializer 返回的字段

```python
python manage.py shell -c "from apps.orders.serializers import OrderSerializer; print(OrderSerializer().fields.keys())"
```

### 7.3 前端 Network 面板排查

1. 打开浏览器 DevTools → Network
2. 访问 `/admin/dashboard`，找到 `/api/admin/stats/` 请求
3. 如果返回 500，查看 Response 体的 `FieldError` 详情
4. 如果返回 200 但数据为 0，检查字段名是否匹配
