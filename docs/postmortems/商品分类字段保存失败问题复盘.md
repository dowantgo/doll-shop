# 商品分类字段保存失败问题复盘

## 1. 结论先行

本次"商品管理"页面新建商品时分类无法保存的根本原因是：

**`ProductSerializer` 未显式声明 `category` 字段的类型，导致前端传入的分类 ID 无法被正确解析为 Category 外键对象，分类字段实际保存为 `null`，前台按分类筛选时无法查到该商品。**

---

## 2. 现象

- 在商品管理页面新建"哈吉米"商品，选择"毛绒玩偶"分类，保存成功
- 商品列表中"哈吉米"的分类列显示为空（其他商品正常显示分类）
- 前台商城点击"毛绒玩偶"分类筛选，"哈吉米"不在结果中
- 手动编辑已有商品修改分类，分类也无法保存

---

## 3. 底层原因

### 3.1 前后端字段契约不一致

| 位置 | 字段 | 说明 |
|------|------|------|
| 前端表单提交 | `category: <category_id>` | el-select 的 v-model 绑定 form.category |
| 前端表格显示 | `category_name` | 后端返回的只读字段 |
| 后端模型 | `category = ForeignKey(Category)` | 外键字段 |
| 后端序列化器 | `category` | **未显式声明字段类型** |

### 3.2 DRF 隐式字段解析失效

`ProductSerializer` 继承自 `ModelSerializer`，理论上 `Meta.fields` 中包含 `category` 时 DRF 会自动推断字段类型。但实际使用中：

```python
# 未显式声明前，DRF 对 category 的处理：
# - 读取时：可以从 FK 对象正确读取（category.name 等）
# - 写入时：前端传入整数 ID，可能无法正确转换为 Category 对象
```

前端提交的数据：
```json
{
  "name": "哈吉米",
  "category": 3,          // 毛绒玩偶的分类 ID
  "price": 99.00,
  "stock": 10
}
```

后端保存时 `category` 字段为 `null`，但没有报错，因为模型定义 `null=True`。

### 3.3 前端表单字段名修复后的遗留问题

在上一轮修复中，前端已从 `category_id` 改为 `category`，与后端字段名对齐。但后端 serializer 仍然缺少显式的 `PrimaryKeyRelatedField` 声明，导致写入路径不通。

---

## 4. 修复动作

### 4.1 后端序列化器修复

**文件**：`apps/products/serializers.py`

```python
class ProductSerializer(serializers.ModelSerializer):
    images = ProductImageSerializer(many=True, read_only=True)
    category_name = serializers.CharField(source='category.name', read_only=True)
    # 显式声明 category 字段类型，确保前端传入 ID 能正确解析为外键对象
    category = serializers.PrimaryKeyRelatedField(
        queryset=Category.objects.all(),
        allow_null=True,
        required=False
    )

    class Meta:
        model = Product
        fields = ['id', 'name', 'description', 'category', 'category_name', 'price', 'cost',
                  'stock', 'sales', 'is_hot', 'hot_sort_order', 'status', 'images', 'created_at']
        read_only_fields = ['id', 'sales', 'created_at']
```

### 4.2 前端表单字段统一

**文件**：`ProductManageView.vue`

| 位置 | 修复前 | 修复后 |
|------|--------|--------|
| el-select v-model | `form.category_id` | `form.category` |
| 验证规则 | `category_id: [{ required }]` | `category: [{ required }]` |
| form 初始值 | `category_id: ''` | `category: null` |
| handleEdit 回填 | `category_id: row.category?.id` | `category: row.category` |
| 表格列 | `prop="category.name"` | `prop="category_name"` |

---

## 5. 问题定性

| 维度 | 说明 |
|------|------|
| 问题类型 | 序列化器字段类型未显式声明，导致写入路径不通 |
| 影响范围 | 商品新增/编辑的分类保存、前台分类筛选 |
| 严重程度 | 高（商品分类功能完全失效） |
| 根因 | DRF ModelSerializer 隐式推断在某些场景下不可靠，需显式声明外键字段 |

---

## 6. 防复发清单

1. **序列化器中所有外键字段必须显式声明**
   - 使用 `PrimaryKeyRelatedField` 或 `SlugRelatedField` 明确指定如何解析传入值
   - 不要依赖 ModelSerializer 的隐式推断

2. **前后端字段名必须完全一致**
   - 前端提交字段名 = 后端 serializer 字段名 = 模型字段名
   - 本项目中统一使用 `category`（不是 `category_id`）

3. **外键字段必须指定 allow_null=True**
   - 模型中 `null=True` 允许为空，serializer 中也需一致
   - 否则空值传入时会报 NOT NULL 约束错误

4. **新建商品后验证数据库实际值**
   - 通过 Django Admin 或 shell 检查商品 category_id 是否正确保存
   - 发现 null 值及时排查 serializer

5. **前台分类筛选前确保 status=true**
   - 模型定义 `status=True` 为上架状态
   - 前台筛选接口会叠加 status 条件，新商品默认上架应能查到
