# 秒杀后台管理模块-面试沉淀

## 1. 为什么要把“评价审核”替换成“秒杀管理”？
- 业务优先级变化：第二次迭代主目标是提升转化率与活动运营效率，秒杀属于直接转化模块。
- 架构拆分更清晰：评价是用户内容域，秒杀是营销交易域；后台入口按业务域拆分更符合职责边界。
- 管理动作可观测：秒杀活动需要频繁调库存、调价格、改状态，必须有日志和回溯能力。

## 2. 本次后台秒杀管理实现了什么？
- 活动管理：新增/编辑/删除秒杀活动。
- 运营动作：库存调整、价格调整、状态切换（draft/online/ended）。
- 风险兜底：预占记录查询与人工释放。
- 可审计性：管理员动作日志（谁在何时改了什么）。
- 运营视图：活动数、在线数、预占与支付转化率等统计。

## 3. 关键设计点与取舍
- 权限模型：所有接口统一 `IsAdminUser`，避免普通用户误调用。
- 一致性：库存变更与预占释放使用事务和行锁，避免并发写冲突。
- 不变量保护：`total_stock` 不允许小于 `reserved_stock`。
- 状态切换保护：活动过期后不可强制上线，减少运营误操作。
- 审计闭环：对 create/update/delete/adjust/change/release 全量落日志。

class AdminSeckillActivityViewSet(ModelViewSet):
    permission_classes = [IsAdminUser]

    def adjust_stock(activity_id, mode, quantity):
        activity = get_activity(activity_id)
        target = calc_target(activity.total_stock, mode, quantity)
        if target < activity.reserved_stock:
            raise BadRequest("total stock < reserved stock")
        before = snapshot(activity)
        activity.total_stock = target
        activity.save()
        write_action_log(action="adjust_stock", before=before, after=snapshot(activity))

    def adjust_price(activity_id, seckill_price):
        activity = get_activity(activity_id)
        before = snapshot(activity)
        activity.seckill_price = seckill_price
        activity.save()
        write_action_log(action="adjust_price", before=before, after=snapshot(activity))

    def change_status(activity_id, status):
        activity = get_activity(activity_id)
        validate_status_transition(activity, status)
        before = snapshot(activity)
        activity.status = status
        activity.save()
        write_action_log(action="change_status", before=before, after=snapshot(activity))

    permission_classes = [IsAdminUser]

    @action(detail=True, methods=["patch"])
    def release(reservation_id):
        with transaction.atomic():
            reservation = lock_reservation(reservation_id)
            if reservation.status != "reserved" or reservation.order_id:
                raise BadRequest("only reserving and un-ordered reservation can be released")
            before = snapshot(reservation)
            release_reservation_quota(reservation, new_status="cancelled", restore_product_stock=True)

## 6. 典型面试追问
- 问：为什么还要“人工释放预占”？
- 答：自动过期兜底只能处理时间窗问题，人工释放用于处理异常支付、风控拦截、运营误配等业务事故，属于运维级兜底能力。

- 问：动作日志里为什么保留 before/after？
- 答：可做完整审计和回滚评估。仅记录“谁做了什么”不足以还原现场，before/after 才能做差异核对。

- 问：如果后续接入秒杀审批流怎么办？
- 答：在 `change_status` 前增加工作流状态机（待审批->已审批->上线），并把审批动作同样写入 `SeckillAdminActionLog`，复用现有审计链路。
