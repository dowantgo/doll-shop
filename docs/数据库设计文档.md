# 数据库设计文档

- 文档版本: v1.1
- 更新时间: 2026-04-09
- 数据库: MySQL 8+
- ORM: Django ORM

## 1. 设计原则
1. 业务主数据落 MySQL，缓存与验证码落 Redis。
2. 订单与支付相关写操作使用事务保证一致性。
3. 通过外键约束维护实体关系。
4. 热点查询字段建立索引（如订单状态、商品上下架、用户订单时间）。

## 2. 实体关系总览

```text
User 1---N Address
User 1---N Order
User 1---N CartItem

Category 1---N Product
Product 1---N ProductImage
Product 1---N InventoryLog
Product 1---N CartItem
Product 1---N OrderItem

Order 1---N OrderItem
Order 1---N PaymentTransaction
Order N---1 Address
Order N---1 User
```

## 3. 核心表说明

### 3.1 users_user
- 用途: 系统用户（继承 AbstractUser）
- 关键字段:
- `id` 主键
- `username` 唯一
- `email`
- `phone`
- `role` (`user` / `admin`)
- `avatar`
- `created_at`, `updated_at`

### 3.2 users_address
- 用途: 收货地址
- 关键字段:
- `id` 主键
- `user_id` 外键 -> `users_user.id`
- `name`, `phone`
- `province`, `city`, `district`, `address`
- `is_default`
- `created_at`, `updated_at`

### 3.3 products_category
- 用途: 商品分类
- 关键字段:
- `id` 主键
- `name` 唯一
- `description`
- `icon`
- `sort_order`
- `created_at`

### 3.4 products_product
- 用途: 商品主表
- 关键字段:
- `id` 主键
- `name`, `description`
- `category_id` 外键 -> `products_category.id`（可空）
- `price`, `cost`
- `stock`, `sales`
- `is_hot`, `hot_sort_order`
- `status`（上架状态）
- `created_at`, `updated_at`

- 索引:
- `(is_hot, hot_sort_order)`
- `(status)`

### 3.5 products_productimage
- 用途: 商品图片
- 关键字段:
- `id` 主键
- `product_id` 外键 -> `products_product.id`
- `image`
- `is_main`
- `sort_order`
- `created_at`

### 3.6 products_inventorylog
- 用途: 库存日志
- 关键字段:
- `id` 主键
- `product_id` 外键 -> `products_product.id`
- `log_type` (`in/out/adjust/return`)
- `quantity`
- `before_stock`, `after_stock`
- `remark`
- `created_at`

### 3.7 cart_cartitem
- 用途: 购物车项
- 关键字段:
- `id` 主键
- `user_id` 外键 -> `users_user.id`
- `product_id` 外键 -> `products_product.id`
- `quantity`
- `created_at`, `updated_at`

- 约束:
- 联合唯一: `(user_id, product_id)`

### 3.8 orders_order
- 用途: 订单主表
- 关键字段:
- `id` 主键
- `order_id` 业务单号（唯一）
- `user_id` 外键 -> `users_user.id`
- `address_id` 外键 -> `users_address.id`（`SET NULL`）
- `total_price`
- `status` (`pending/paid/shipped/delivered/cancelled`)
- `payment_status` (`pending/paid`)
- `payment_method` (`alipay/wechat/mock`)
- `shipping_company` (`sf/zto/yto/sto/yunda`)
- `shipping_status` (`not_shipped/shipped/in_transit/arrived/signed`)
- `tracking_no`, `trade_no`
- `remark`
- `created_at`, `paid_at`, `shipped_at`, `delivered_at`, `expires_at`, `updated_at`

- 索引:
- `(user, -created_at)`
- `(status)`
- `(payment_status)`

### 3.9 orders_orderitem
- 用途: 订单商品项
- 关键字段:
- `id` 主键
- `order_id` 外键 -> `orders_order.id`
- `product_id` 外键 -> `products_product.id`（可空）
- `quantity`
- `price`（下单时快照价）
- `created_at`

### 3.10 payment_transaction
- 用途: 支付交易单（当前支付主模型）
- 关键字段:
- `id` 主键
- `order_id` 外键 -> `orders_order.id`
- `payment_method` (`alipay/wechat/mock`)
- `trade_no`（三方交易号，唯一，可空）
- `out_trade_no`（商户交易号，唯一）
- `amount`
- `status` (`pending/paid/expired/closed/failed/refunded`)
- `qr_code`
- `expire_time`, `paid_time`
- `created_at`, `updated_at`

- 表名说明:
- 该模型显式指定 `db_table = payment_transaction`

## 4. 事务与一致性

### 4.1 下单事务
1. 校验购物车与地址
2. 锁定商品库存（`select_for_update`）
3. 创建订单与订单项
4. 扣减库存并增加销量
5. 清空购物车

### 4.2 取消订单事务
1. 校验订单可取消
2. 锁定相关商品
3. 回补库存并回退销量
4. 更新订单状态

### 4.3 支付事务
1. 创建支付时锁定订单与待支付交易
2. 回调/查询支付成功时事务更新交易与订单

## 5. 缓存与非持久数据

### 5.1 Redis 键
- `captcha:{captcha_id}` 图片验证码
- `email_code:{type}:{email}` 邮箱验证码

### 5.2 过期策略
- 图片验证码: 5 分钟
- 邮箱验证码: 10 分钟

## 6. 历史兼容说明
1. `users.PaymentRecord` 为历史支付记录模型，当前支付主流程使用 `payment.PaymentTransaction`。
2. 后续建议统一支付账务模型，减少双模型维护成本。

## 7. 迁移与维护建议
1. 所有结构变更必须走 Django migration。
2. 生产环境执行 migration 前先备份数据库。
3. 对高频查询接口按慢查询日志持续补索引。
